# Docker 容器启动顺序与监控管理工具 - 使用文档

## 概述

这是一个现代化的Web界面工具，用于管理飞牛NAS（或其他Docker环境）中的容器启动顺序、定时健康检查和自动重启。整个系统采用极简、自然的设计风格，提供直观的操作体验。

## 主要功能

### 1. 仪表盘（首页）

仪表盘展示了容器管理系统的整体概览：

- **统计卡片**：显示总容器数、已启用监控的容器数、未启用监控的容器数
- **容器列表**：以卡片形式展示所有受管容器的配置信息
- **最近日志**：实时显示系统的运行日志，包括启动事件、重启记录和错误信息
- **添加容器**：通过右上角"添加容器"按钮快速添加新的受管容器

### 2. 启动顺序管理

在"启动顺序"页面，您可以：

- **查看启动顺序**：容器按照配置的顺序显示，每个容器前有顺序号
- **拖拽排序**：点击并拖拽容器卡片可以调整启动顺序
- **编辑配置**：点击"编辑"按钮修改容器的启动延迟和监控开关
- **删除容器**：点击"删除"按钮移除不需要的容器配置
- **保存更改**：编辑完成后点击"保存"按钮应用更改

### 3. 全局设置

在"设置"页面，您可以配置系统级别的参数：

- **健康检查间隔（check_interval）**：设置系统检查容器状态的时间间隔（单位：秒，最小值：5秒，推荐值：60秒）
- **工作原理说明**：详细说明系统如何启动容器和监控状态
- **日志说明**：解释不同类型的日志事件

## 工作流程

### 系统启动

1. 系统启动时，自动按照容器列表中的顺序依次启动容器
2. 每个容器启动后，系统等待其配置的延迟时间（startup_delay）再启动下一个
3. 所有容器启动完成后，系统进入监控模式

### 监控模式

1. 系统每隔 `check_interval` 秒检查一次已启用监控的容器
2. 对于每个被监控的容器：
   - 如果容器正在运行，记录"运行中"状态
   - 如果容器已停止，系统尝试自动重启，并记录重启事件
   - 如果容器未找到，记录错误信息
3. 所有操作都会被记录到日志中

## 容器配置说明

每个容器配置包含以下参数：

| 参数 | 类型 | 说明 |
|------|------|------|
| 容器名称 | 字符串 | Docker容器的实际名称 |
| 启动顺序 | 数字 | 容器启动的顺序（0表示第一个启动） |
| 启动延迟 | 数字 | 容器启动后等待多少秒再启动下一个（单位：秒） |
| 启用监控 | 布尔值 | 是否对该容器进行定时健康检查和自动重启 |

## 日志说明

系统记录的日志分为以下几种类型：

| 日志类型 | 说明 | 示例 |
|---------|------|------|
| 启动 | 容器启动事件 | "容器启动成功" |
| 重启 | 自动重启事件 | "容器已自动重启" |
| 检查 | 定时状态检查 | 系统定期检查容器状态 |
| 错误 | 错误事件 | "容器启动失败"、"容器未找到" |

## 容器状态说明

系统识别以下三种容器状态：

- **运行中**：容器正在运行，系统正常工作
- **已停止**：容器已停止，如果启用了监控，系统会尝试重启
- **未找到**：容器在Docker中不存在，可能已被删除

## 最佳实践

### 1. 设置合理的启动延迟

- 对于依赖其他服务的容器（如应用程序），应该在数据库容器之后启动
- 设置适当的延迟时间，确保前置服务完全启动
- 示例：MySQL (0s) → Redis (5s) → 应用程序 (10s)

### 2. 选择合适的监控间隔

- 对于关键服务，建议使用较短的检查间隔（30-60秒）
- 对于非关键服务，可以使用较长的间隔（120-300秒）
- 避免设置过短的间隔，以免对系统造成过大负担

### 3. 监控开关的使用

- 对于必须持续运行的容器，启用监控
- 对于临时或按需启动的容器，禁用监控
- 定期检查日志，确保系统正常运行

## 故障排查

### 容器无法启动

1. 检查容器名称是否正确
2. 查看日志中的错误信息
3. 确保Docker服务正常运行
4. 检查容器是否存在

### 监控不工作

1. 确认容器的"启用监控"开关已打开
2. 检查全局设置中的检查间隔是否过长
3. 查看日志中是否有错误信息
4. 重启应用程序

### 日志为空

1. 确认系统已启动
2. 检查是否有容器配置
3. 等待一个检查周期（根据check_interval设置）

## 部署说明

### 使用Docker Compose部署

```yaml
version: '3'
services:
  docker-manager-ui:
    image: docker-manager-ui:latest
    container_name: docker-manager-ui
    restart: always
    ports:
      - "3000:3000"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - docker-manager-db:/data/db
    environment:
      - DATABASE_URL=mysql://user:password@host:3306/docker_manager
      - TZ=Asia/Shanghai
    networks:
      - docker-manager-network

volumes:
  docker-manager-db:

networks:
  docker-manager-network:
    driver: bridge
```

### 环境变量配置

- `DATABASE_URL`：数据库连接字符串
- `TZ`：时区设置（默认：Asia/Shanghai）

## 常见问题

**Q: 如何重置所有配置？**
A: 删除数据库中的所有容器配置记录，然后重新添加。

**Q: 系统支持多少个容器？**
A: 理论上无限制，但建议不超过100个以保证性能。

**Q: 能否手动触发启动序列？**
A: 目前通过API支持，后续版本将在UI中添加一键启动按钮。

**Q: 日志保留多久？**
A: 默认保留最近100条日志，可在API中修改limit参数。
